Using parameters

You can easily configure PhpMetrics according to your needs.

--config=<file.yml>
Config file to use (YAML)
--failure-condition=<expression>
Optional failure condition, in english. For example: average.maintainabilityIndex < 50 or sum.loc > 10000
--extensions=<php|inc>
Regex of extensions to include
--excludedDirs=<tests|Tests|Features>
Regex of subdirectories to exclude
--report-html=<file.html>
Path to save report in HTML format
--report-xml=<file.xml>
Path to save report in XML format
--report-csv=<file.csv>
Path to save report in CSV format
--violations-xm=<file.xml>
Path to save violations in XML format. Same format as PHPMessDetector
--level=<0>
Subdirectories to explore. 0=infinite (default)
--without-oop
Disable all OOP metrics. May be unstable

Using configuration file

You should use the --config=<file.yml> option.

Here is an example of configuration file.

{% highlight yaml %} default: rules: cyclomaticComplexity: [ 10, 6, 2 ] failure: average.maintainabilityIndex < 50 or sum.loc > 10000 path: directory: src extensions: php exclude: Features|Tests logging: report: xml: ./log/phpmetrics.xml html: ./log/phpmetrics.html csv: ./log/phpmetrics.csv violations: xml: ./log/violations.xml {% endhighlight %}
